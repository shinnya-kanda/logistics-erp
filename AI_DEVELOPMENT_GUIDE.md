
# AI_DEVELOPMENT_GUIDE.md
Logistics ERP – AI Development Guide
Author: Shinya Kanda
Date: 2026-03-17

This document explains how to use AI tools (ChatGPT, Cursor, Claude)
to efficiently develop the logistics-erp project.

The purpose is to make AI understand the project context and accelerate development.

This file should be placed in the repository root.

--------------------------------------------------
1. Required Context Files
--------------------------------------------------

Before asking AI to implement anything, always provide context.

Required files:

MASTER_CONTEXT.md
PROJECT.md
docs/db-schema.md
docs/roadmap.md

These files explain:

• project philosophy
• architecture
• database design
• development phases

Without these files AI may generate incorrect designs.

3. Standard Development Workflow
--------------------------------------------------

Step 1

Implementation with Cursor

Architecture review with Claude

Example:

"Review this design according to MASTER_CONTEXT.md principles."

--------------------------------------------------
7. Development Rules
--------------------------------------------------

Important principles:

1. Implement features phase by phase.
2. Avoid building the full ERP at once.
3. Maintain Expected vs Actual separation.
4. Keep services modular.
5. Use migrations for schema changes.
6. Avoid quick hacks in database structure.

9. Key Architectural Principle
--------------------------------------------------

The most important concept of this project:

Expected vs Actual

Expected

Shipment plans created from PDF instructions.

Actual

Operational scan events generated by drivers and warehouse workers.

The system compares both to detect shipment errors.
